# Phase B — 校准 + 图作先验融合(真实未来判据)

> 日期:2026-07-02。基于 [phase_a_real_future_20260702.md](phase_a_real_future_20260702.md)。
> 所有数字在 held-out episode 划分(40,572 pairs / 611 episodes / 37 milestones)上,对**真实观测**的未来 milestone 评分。

## 问题框架

Phase A 证明真实未来贪心头已是一个帧条件的下一 milestone 分布(softmax 以 CE 训练于观测未来),且其 NLL 反超经验基线。Phase B 不重新训练,问两个诚实问题:

1. **校准** —— 预测置信度可信吗?能否通过温度缩放修复?
2. **图作先验** —— 将图从标签/硬回退降级为软贝叶斯先验,通过对数线性池化
   `log p = (1-lam)·log p_neural + lam·log p_prior`,检查是否对真实未来 NLL / top-k 有帮助。
   `p_prior = transition_probs[current_m]`(经验 milestone 级行)。

脚本:`scripts/eval_phase_b.py`。产物:`outputs/phase_b_eval/realfuture/summary.json`。

## 1. 校准

| 指标 | 值 |
|---|---|
| ECE(原始) | 0.103 |
| Brier | 0.773 |
| 温度 `T`(calib 半拟合) | 1.30 |
| test ECE 前→后 | 0.097 → **0.005** |
| test NLL 前→后 | 1.961 → 1.916 |

可靠性分箱显示轻度一致的过自信(如 0.5-0.6 箱中,平均置信度 0.547 vs 准确率 0.425)。单个温度 `T=1.30` 就基本消除了全部偏差(ECE 0.097→0.005,约 20 倍)。**帧条件分布在单参数修复后可信。**

## 2. 图作先验对数线性融合(对真实未来)

| lam | top1 | top3 | top5 | NLL |
|---|---|---|---|---|
| 0.0(仅神经) | 0.383 | 0.686 | 0.822 | 1.978 |
| 0.2 | 0.416 | 0.731 | 0.855 | 1.812 |
| **0.3(最优 NLL)** | **0.417** | **0.731** | **0.856** | **1.798** |
| 0.4 | 0.415 | 0.727 | 0.854 | 1.809 |
| 0.5 | 0.409 | 0.720 | 0.847 | 1.844 |
| 1.0(仅先验) | 0.240 | 0.483 | 0.633 | 2.572 |

曲线光滑凹,有明显的最优点 **lam ≈ 0.3**:约 30% 图先验将 top1 从 0.383 提升到 **0.417**,top5 从 0.822 到 **0.856**,NLL 从 1.978 到 **1.798**。

## 发现

1. **分布可信。** 原始 ECE 0.10→0.005(T=1.30);经温度缩放后置信度可用于下游门控。
2. **图作软先验确实有帮助 —— 这次是诚实的。** 在 `lam=0.3` 时,图改善了每个真实未来指标。这是旧硬回退的原则性替代:旧的 hybrid"0.997"是对定义其标签的图验证的(循环);本次融合对现实验证且增益真实但温和(top1 +3.4pt,NLL −0.18)。
3. **单用任一组件都不够。** 仅先验(0.240/2.572)远差于仅神经(0.383/1.978);融合超过两者。帧特征和 milestone 级先验携带互补信息。

## 推荐 real-future 推理配方

```
p = softmax(logits / 1.30)^0.7 · P(next | cur_milestone)^0.3
```

汇报 top-k + NLL,不用 top1-vs-graph。这取代了图查找 hybrid 门控,用于任何须反映真实下一 milestone 预测的评估。

## 诚实说明

- 绝对 top1 ≈ 0.42 反映真实的 ~13 分支问题;top-3/top-5(0.73/0.86)和 NLL(1.80)是有意义的数字。
- `T` 和 `lam` 在 held-out `kai0_base` 上调优 —— 每数据集/划分需重新拟合。
- 仍为单 `kai0_base` DINOv3-H,同任务 held-out,隐变量 prototype 输出。
