# LMWM-2 最终定档架构(2026-07-08 数据收敛)

> 从 LMWM-2 重设计方案出发,经 P1/P2 两轮 intrinsic 探针(本地 2×A100,3 任务 12k step,DINO 沙箱 8000 步),**每一个设计赌注都经数据裁决后定档**。本文是最终权威版,取代 `REDESIGN_LMWM2_2026-07.md` 的待定部分。后续 P0(oracle-SR)以此架构为基座。

---

## 1. 定档架构总览

```
部署(在线,零 CRAVE / 零 DINO / 零 bank,~34M):
  当前帧 224² ──π0.5 SigLIP(冻结,同塔)──→ grid G_t(16²×1152) ──pool──→ gist g_t(1152)
  上一步码 prev_ẑ(128,模型自身上次输出)

  ① 预测器 Predictor(MDN K=4,3.5M)
     输入: g_t ⊕ prev_ẑ ⊕ proprio
     输出: 候选码 {ẑ_k} + 密度 {logp_k}
     ↓
  ② 候选选择(零参数几何 + 模型自身密度):
     score_k = logp_k(MDN 自身密度,数据裁决:替代死掉的熵门,AUC 0.62)
             × 𝟙[u·(ẑ_k − z_t) > δ](前向锥,弱但零成本)
     全否/全低密度 → 弃权:本步不注入,交还 π0.5 语言通道
     ↓
  ③ 生成器 Generator(AdaLN CNN 四块,30M,原样保留)
     输入:(G_t 画布, ẑ* 调制码)
     输出: 子目标 Ĝ_sub(16²×1152)
     ↓
  ④ 注入 π0.5(接线复用 lmwam_v2 plan):
     Ĝ_sub 下采样→prefix 新视觉槽 + 零初始化投影 + KI stop-grad + 去同步增广

调度(事件驱动):
  达成: ‖z_t − ẑ*‖ < τ → prev_ẑ = ẑ*,重预测下一 milestone(天然滞回)
  停滞: u·Δz_t ≈ 0 超 τ_s → 重选候选重发
  弃权: 密度门 → 退语言通道
```

## 2. 逐设计赌注的裁决表

| # | 赌注 | 探针 | 数值 | 定档 |
|---|---|---|---|---|
| 1 | **价值=方向 u·z**(单码空间承载价值) | mono u·z vs gist | 0.557 < 0.612 | ❌ 证伪 → 价值退独立头(~0.2M);单帧进度本身有天花板(0.648) |
| 2 | **MDN pi 熵门放弃权** | entropy / ECE / gate@0.9 | 0.0003 / 0.637 / 0 | ❌ 证伪(pi 塌成 one-hot) → 改用密度弃权(logp);密度分对错 AUC ~0.63 |
| 3 | **MDN 自身密度作 verifier**(无需外挂 head) | verify AUC 倒退/错任务 | 0.857 / 0.996 | ✅ 保留:优雅成立,无需外挂 verifier 网络 |
| 4 | **码距达成判定** | reach AUC 码距/学习头 | 0.575 / 0.651 | ⚠️ 码距弱,学习头更好 → **两档切换**:先码距(AUC>0.6 则用,零参数),不掉则学习头(~0.1M) |

| 7 | **检索 payload** | 用户定案 | — | ❌ 否决:锁死历史记忆 |
| 8 | **预测器 CNN vs ViT / diffusion 头** | 自家 H7 / flow < 回归 | 空间 token 无增益 / flow deploy 反降 | ❌ 否决:保持 CNN MLP 预测器 + MDN 头 |
| 9 | **单码空间承载身份**(proto teacher) | deploy/id3 vs proto 定档 | 0.7564/0.7098 ≈ 0.7532/0.710 | ✅ 保留 |

## 3. 各组件 I/O 规范(部署,冻结常量随 ckpt)

| 组件 | 参数 | 输入 → 输出 | 定档来源 |
|---|---|---|---|
| ⓪ π0.5 SigLIP | ~400M 冻结 | 帧 224²×3 → grid G_t(16²×1152) | 同塔复用 KV |
| gist 池化 | 0 | G_t → mean(2,3) → g_t(1152) | — |
| z_t 投影 | W(1152×128),~0.15M | g_t → z_t(128) | proto 谱系 |
| ① 预测器 MDN | ~3.5M | g_t⊕prev_ẑ(128)⊕proprio(~32) → π(4)+μ(4×128)+σ(4×128) | predm 谱系 + prev_ẑ 条件(数据裁决 P2) |
| ② 候选选择 | 0(几何+密度) | {ẑ_k, logp_k} → ẑ* / 弃权 | 密度弃权(数据裁决,替代熵门) + 前向锥 |
| ③ 生成器 AdaLN | 30M | (G_t 16²×1152, ẑ* 128) → Ĝ_sub(16²×1152) | 原样继承(消融已定档) |
| 价值头(测试时门控) | ~0.2M | g_t → v∈[0,1] | 数据裁决:u·z 证伪后回退;仅软门控 |
| u 方向(前向锥) | 128,~0.0001M | ẑ_k, z_t → u·(ẑ_k−z_t) | weak 但零成本,保留作辅助信号 |
| 达成判定 | 0 或 ~0.1M | z_t, ẑ* → reached bool | 码距优先;不够切学习头(预案 3) |
| ④ 注入投影 | ~0.3M(零初始化) | Ĝ_sub → π0.5 prefix 2048 | lmwam_v2 接线复用 + KI stop-grad |

**部署总参数: ~34.5M** (① 3.5M + ③ 30M + 投影 ~0.65M + W/u ~0.15M + 价值头 ~0.2M)。2 网络 + 几何运算,零查表零索引。

## 4. 跨任务机制

1. **连续码空间**(proto 谱系):身份活在 SigLIP 连续流形,LOO 已证连续锚 unseen 小胜 → 开放词表;
2. **prev_ẑ 自递归条件**(数据裁决 P2 主杠杆):条件化增益来自**上一 milestone 的码**,与自己输出的反馈循环天然跨任务;
3. **生成器画布机制**:外观从当前观测来,对 unseen 场景零依赖;
4. **语言条件**(T7):当前单指令数据集无增益(P2),但不代表多任务混合指令无增益→ P3 在 action-MAE 下复测;
5. **proto teacher 训练期共享**:簇中心码蒸馏天然多任务。

## 5. 编码器耦合策略

**前提**:先查 kai0 π0.5 后训练是否冻结 SigLIP 塔 → 若冻结,问题消失,直接复用。
**若塔漂移**:核心 KI 冻死(保护脆弱的价值/身份结构),塔依赖塞进薄适配头(当前塔→LMWM 规范空间);漂移时自监督重拟合薄壳(便宜、无 SR 标签)。"联合"降级为只训接口(适配头+注入投影),核心永远冻结。纯全量联合仅留作 P0 SR 打不出增量的最后逃生舱。

## 6. 训练(离线)

CRAVE per-task 挖矿 → proto teacher 出码 z → 双损失:
- **L_proto**: MDN NLL 蒸馏 z(身份监督)
- **L_gen**: smooth-L1(Ĝ_sub, G_target) + lift(cos 相对距离,防塌回 persist)
- prev_ẑ 条件: teacher-forced(训练期用 GT 上一 milestone 码);部署时用自己的上一步输出
- bank 质量验收三关卡(簇纯度/value 跨度/身份可分度)

CRAVE/DINOv3-H/**bank 仅训练期存在,随 ckpt 交付的只有 W/u/归一化统计(类 BN 参数)。**

## 7. 降级阶梯(仅剩两条未裁决)

| 机制 | 优雅方案 | 实测状态 | 降级路径 |
|---|---|---|---|
| 达成 | 码距阈值 τ | AUC 0.575(边缘) | → 学习 reach 头(~0.1M)(已训对照,AUC 0.651) |
| 价值 | u·z 方向 | 0.557(证伪) | → **已降级**:独立价值头(~0.2M) |

—— 两条降级都只加参数、不碰红线(学习式、零 CRAVE);已落地的降级(价值头)已加入 §3 的 I/O 表。
