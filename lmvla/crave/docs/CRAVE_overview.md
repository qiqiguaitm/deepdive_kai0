# CRAVE: 基于共现状态聚类的无监督任务价值估计

## —— 方法、架构与效果总览

> **日期**: 2026-06-29  
> **版本**: v2.4  
> **关键词**: DINOv3-H · 共现聚类 · milestone · Viterbi 解码 · 零训练跨域迁移

---

## 一、核心思想

CRAVE (Clustering-based Recurrence Analysis of Visual Episodes) 的核心洞察很简单:

> **跨 episode 反复出现的视觉状态 = 任务必经结构。**

同一任务的大量示范视频中,关键阶段(如"手接近衣领"、"对折衣服"、"叠好完成")会在不同 episode 中反复出现。CRAVE 用无监督聚类自动发现这些重复出现的关键状态(milestone),并给每个状态分配一个 0→1 的稠密进度值(value)。

整个过程**不需要任何人工标注、不需要训练神经网络**,纯数据驱动。

### 2.1 特征编码层

**视觉编码器**: DINOv3-H (ViT-H/16+, 1280 维 pooled 特征)
- 编码分辨率: 224×224
- 精度: bf16
- 环境: srpo conda env

**本体状态融合**: 28D 关节位置 (qpos)
- 归一化后以权重 0.2 拼接至视觉特征
- 最终特征: 1280 + 28 = **1308D**

**预处理**:
1. L2 归一化
2. 时序平滑 (窗口=3)
3. 3Hz 子采样 (从 30Hz 原始视频每 10 帧取 1 帧)

**缓存**: 全量特征缓存于 `temp/crave_full_dinov3h/` (30 万+ 帧)

### 2.2 共现聚类层

使用 MiniBatchKMeans, 初始聚类数 K₀ = 120:

- K₀=120 确保过分割(每个"真实"任务阶段被拆为 ≈ 3-5 个子簇)
- 后续用覆盖率+纯度筛选合并为 milestone

聚类在 **1308D 联合空间**中进行, 不仅靠视觉相似度, 还包含了任务共现模式——"手在领口上方"和"手在衣角位置"被分到不同簇不仅因为像素不同, 更因为它们在任务中出现的模式不同。

### 2.3 Milestone 筛选层

三个标准共同筛选出最终 milestone:

1. **覆盖率**: 该簇覆盖的 episode 比例。覆盖 >50% = 该状态在大多数示范中出现 → 是任务的必经结构
2. **纯度**: 簇内 value 的标准差应低于中位值 (<0.09)。高纯度 = 该簇内的帧进度一致
3. **均匀性**: 用差分进化算法从候选簇中选择均匀覆盖整个任务进度的子集 (M ≈ 40-80)

### 2.4 价值解码层

解码阶段用四种机制确保 value 平滑可解释:

**① 进度先验 (emit_of)**
- 每个 milestone 的"理想进度" = 该簇所有帧的 value 均值
- 赋予高斯置信: γ=0.8, 离簇均值越远置信越低
- 远处进度的"远前方保护": γ_fwd=1.3, 防止后半段提前高

**② Viterbi 解码 (viterbi_pen)**
- 转移惩罚系数 λ_geo=1, 鼓励单调前进
- 向后屏障 β=0.4: 倒退一步扣 40% 置信
- 末端硬屏障: 接近完成时不能大幅回到起点

**③ 置信度门控 (conf_hold)**
- 只有当最近 milestone 距离 < 阈值时才置信
- 跨域迁移时自动使用目标域自身的距离分布百分位数

**④ 单调平滑 (smooth_monotone)**
- 强制最终 value 全局单调非递减
- 修复 Viterbi 残留的局部乱抖

## 三、训练与推理

### 训练 (零训练!)

CRAVE **不需要梯度训练**。整个流程是:

| 步骤 | 需要 GPU? | 耗时 (30 万帧) |
|------|-----------|---------------|
| DINOv3-H 特征提取 | ✅ (2×A100, bf16) | ~8 小时 |
| MiniBatchKMeans 聚类 | ❌ (纯 CPU) | <5 分钟 |
| Milestone 筛选 | ❌ | <1 分钟 |
| Viterbi 解码 | ❌ | <1 分钟 |

核心成本是一次性 DINOv3-H 编码。聚类和解码都是 CPU 秒级完成。

### 推理

对新 episode:
1. DINOv3-H 编码 + proprio 拼接 → 1308D 特征
2. KMeans 分配到最近簇
3. Viterbi 解码 → 平滑 value

**跨域推理**: 不需要重聚类、不需要微调。用源域挖好的 milestone, 使用目标域自身的距离分布重新计算置信度阈值即可。

## 四、关键效果

### 4.1 零训练直接解掉传统价值模型的三大痛点

| 痛点 | 传统方法 | CRAVE |
|------|---------|-------|
| **① 抖动** | value 每帧剧烈摇摆 | **单调平滑**, mono≈0.8→0.98 |
| **② 不可解释** | 价值升降原因不明 | 每次升降可指着 **具体 milestone** 说明 |
| **③ 末端假跌** | 任务末尾 value 异常跌落 | **转移概率**压住末端→起始跳转 |

### 4.2 跨域泛化 (零迁移)

| 目标域 | 配置 | 相关性 |
|-------|------|--------|
| KAI0 kai0_base (同域, holds-out) | 逐字不改 | corr > 0.96 |
| **XVLA** (全新本体/相机/布料) | 逐字不改 | **corr 0.956** |
| **ALOHA 咖啡** (互联网数据,完全未知) | 逐字不改 | **corr 0.988** |
| **v4** (新录制环境) | 只在置信度阈值用目标域百分位 | **end=0.87, 99% 单调** |

### 4.3 Value 质量指标

- 簇内 value std 中位: **0.09** (同进度帧高度一致)
- 相干簇帧-均值 R²: **0.84** (簇均值忠实代表帧)
- 单调性 (mono): **0.80-0.98** (Viterbi 前→后)
- 跨域 episode 覆盖率: **50-83%** (milestone 确实出现在大多数 episode)

### 4.4 定性效果

- Milestone 可解释: 簇中心解码图与最近真实帧高度一致
- 转移概率图: 对角线右上为主体(任务单向前进), "中段→起始" ≈ 0
- 价值升降诊断: 可精确追溯到具体 milestone 跳变
