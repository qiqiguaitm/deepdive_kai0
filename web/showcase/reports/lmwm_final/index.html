01 / 指标

各指标是什么、表明什么

我们预测"未来某 milestone 的 DINOv3 特征网格"。这些指标从不同角度衡量预测质量 —— 按本科生级直白解释。

指标直白定义作用 / 表明什么
grid-cos预测特征图 与 真实目标图 的余弦相似度(1=一样)核心保真度,越高越准
oracle用"作弊码"(偷看真未来算出的转移码)重建的 cos天花板:知道真答案能做多好 = 模型表达上限
deploy只看当前帧、用预测码重建的 cos实战值:真部署(没未来)能做多好 ← 最关心
persistence把"当前帧"当预测(不动)的 cos白给基线:啥也不预测得多少分
lift= deploy − persistence净技巧:扣掉"目标本来就近"的便宜
model lag预测图匹配到最像的真帧,那帧晚几秒模型"敢预测多远"
ratio= model lag / dataset lag(目标本身跨度)<1 = 欠射(不敢预测那么远,退回当前)
best-of-8VAE 采 8 个预测取最像的 cos未来多不多模态(远超均值=多分支)
corr(value,time)CRAVE 进度值 与 归一化时间 的相关进度标签质量(0.94=很可靠)
monotonicity序列中非递减的比例进度单调不倒退(0.98)

一句话记忆: oracle=上限,deploy=实战,persistence=白给,lift=真本事,ratio=敢多远,best-of-8=多模态。

05 / 结论

三大天花板 · 一个关键杠杆

oracle→deploy 的差距试遍了三条路,都只小幅改善 —— 差距大部分是"从当前预测未来"的硬信息损失。

容量 / Transformer

24M→260M transformer:oracle +0.008,deploy 持平。我们数据规模下 260M 过配。

→ 不是杠杆

多模态 / VAE

best-of-8 随 kl 增长,证实未来弱多模态;但最多回收 +0.02(kl 1e-1)。

→ 小幅

感知端 / 多视角+时序

top_head + 双手腕 + 时序上下文:milestone+1 只 +0.002~0.006。

→ 几乎无用

关键杠杆 = 数据集目标构建。 不是网络内部,而是"预测目标怎么定"。V3.1(CRAVE 进度单调 milestone)让模型敢 commit、跨集一致 —— 这才是真正撬动 deploy 的地方。oracle→deploy 剩余的 ~0.05 是信息论下限,要更准需从根本数据/传感器补。
