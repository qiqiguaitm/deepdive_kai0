01B / 新架构(2026-07 定案)

两模型:身份(MDN)+ 落地 · π0.5 SigLIP 空间

深调研后重构:①多峰在"下一个是哪个 milestone"(身份)而非外观 → Stage-1 用 MDN 多峰分布(回归会塌);②目标空间换成 π0.5 自己的 SigLIP(实测 deploy 0.716 ≥ DINOv3-H 0.694,融合零损失),DINOv3-H+CRAVE 降为离线 label 工厂;③判据从 grid-cos 换成身份 top-N(grid-cos 对身份多峰不敏感)。

03 / 指标

各指标是什么、表明什么

从不同角度衡量"预测得准不准" —— 本科生级直白解释。

指标直白定义作用 / 表明什么
grid-cos预测特征图 与 真实目标图 的余弦相似度核心保真度,越高越准
oracle用"作弊码"(偷看真未来算的码)重建的 cos天花板:知道真答案能做多好 = 表达上限
deploy只看当前帧、用预测码重建的 cos实战值:真部署能做多好 ← 最关心
persistence把"当前帧"当预测(不动)的 cos白给基线:啥也不预测得多少分
lift= deploy − persistence净技巧:扣掉"目标本来就近"的便宜
model lag / ratio预测匹配到的真帧晚几秒 / 除以目标跨度模型敢预测多远;ratio<1 = 欠射
best-of-8VAE 采 8 个取最像的 cos未来多不多模态(远超均值=多分支)
corr(value,time)CRAVE 进度 与 时间 的相关进度标签质量(0.94=很可靠)

04 / 定位

与 LaWM 对照 & 下一步

LaWM(arXiv 2606.15768)官方 checkpoint 在我们数据上实测。⚠️ 编码器空间不同 → 绝对数不硬比,趋势可比。

Transformer(LaWM) CNN(我们) 冻结 不在 LAM 内 · 甩给策略

结构上一句话:LaWM = 重型 Transformer/DiT + 全局注意力 + 小码(32d)+ 预测码甩给策略;我们 = 轻型 CNN + 当前 grid 空间基底 + 内置 predm 自包含 + 更强 ViT-H 编码器。实测:数百 M 的 Transformer-DiT 并未碾压我们 ~27M CNN(oracle 0.770 vs 0.774,lift 相当)。
