问题:单预测器 vs 预测器+生成器

04

机理:两模型的价值 = 多模态,不是单发精度

证否「单模型会持久坍缩」在此设定不成立。A 的 lift +0.100(远 > 0)、copy_cos 0.753 ≈ 自身 recon,不是原地不动。历史坍缩是老编码器/离散标签特有,不普适。

真因分叉点是多模态的。给定当前帧,下一 milestone 常有多个可能分支。单模型只能出一个(条件均值,分叉处必偏);两模型的 MDN 出 K 个分支提议,生成器各渲一张,其中一个命中 → best-of-K 反超。确定性单回归器做不到 best-of-N。

oracle(0.793)证明生成器渲染力足够 —— 真正的瓶颈在预测器从当前帧推断分支。而多模态提议正是绕开它的方式:生成器 + 紧凑 code 的分解 = 多模态子目标提议的使能条件。这才是预测器+生成器架构的正确辩护。

05

诚实的边界(勿过度外推)

  • 内在指标,非下游 SR。recon_cos 到目标 latent ≠ VLA 用它当 hint 后的成功率。按 roadmap 教训(内在 gain 2.1× 曾未换来 SR),此结论不能直接推成「两模型对 VLA 更好」。真判据 = best-of-K 子目标是否提 VLA-SR,待做。
  • So400m/rvalley 特定。老 DINOv3-H/SigLIP 空间可能不同(那里历史观察到坍缩)。
  • best-of-K 需下游能消费多个候选(CFG / 采样 / 重排);若只选一个就退回 B 单发(输于单模型)。
  • A+ 不稳:单模型嫁接 lift 有 seed 塌到 0.673 —— 反持久项在无 code 指向时会过罚。lift 是 v2 的项,单独用脆。
