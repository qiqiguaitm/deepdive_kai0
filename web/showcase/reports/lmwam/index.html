核心 claim

现有 VLA 世界模型用固定时延预测未来帧特征,而在官方实现中这个 τ 被结构性地焊死在动作块最后一帧上,不是可独立调节的超参数。LMWM 用 CRAVE 无监督发现的 milestone 边界替代 τ,把 WM 监督时域与动作块解耦,使预测目标随任务节奏自适应。

痛点 1WM 预测时域与动作块未解耦 中

⚠️ 2026-07-19 更正:本节原为「τ 从未被系统消融」,已证伪并删除。 GR-1(2312.13139) 附录 A.4 表 5 恰恰做了这个消融(CALVIN, Δt∈{1,3,5} → Avg.Len 3.61/3.82/3.67), 而原文点名 GR-1 作为反例。另有 JOPAT(2605.23856 附录 A.3.2)、AHEAD(2606.02486 附录 E.4 表 15)、 OneWM-VLA(2605.07931 §5.6)三篇独立反例;且 SuSIE(2310.10639) 的 τ 是 U[11,14) 区间采样, 连「固定 τ」这个前提都不成立。原先标注为「LaWAM Appendix C.5 原文」的两句引文无法回溯, 实为审计者结论被误当成论文自述,一并删除。

可辩护的表述:少数工作报告过 horizon 敏感性(GR-1、JOPAT),但普遍局限于单一 benchmark、窄范围、 且未与控制频率/物理时长解耦;latent world model 一线(LaWAM 固定 1.2s、Seer n=3、DreamVLA、GR-2) 则一致地把 τ 当作未经检验的实现细节。

主论点(经第一手代码验证):在 LaWAM 官方实现 (lerobot_datasets.py::_sample_video_delta_indices)中,num_frames=2 时未来帧索引 = action_delta_indices[-1] —— 世界模型的预测目标被结构性地焊死在动作块最后一帧上。 因此 τ 不是可独立调节的超参数:想单独消融 τ 就必然同时改动作块长度,二者混淆。 「robot τ=1.2s / egocentric τ=0.4s」实为动作块时长选择(依控制频率定,有正当理由)。 真正的缺口是 WM 监督时域与动作块的解耦 —— 这正是 LMWM 在做的事。
★ 新证据(2026-07-19,24 路变 seed):收益来自"目标选在哪",不是"看多远"
t6(唯一确证有收益的任务)的 milestone 前瞻中位数恰好是 7 帧,与 LaWM 固定的 t+7 基本相同, 且 t6 是全表 milestone 最稀疏(3.9/ep)、episode 最短(49)的任务。 在前瞻距离大致相等的条件下,LMWM 仍赢 +8.5(t≈6.1, p<0.001),而 LaWM 只有 +2.0(ns)。

⇒ 这否定了"LMWM 好在提供更长程指引"这一此前的解释,也意味着 单纯加长 horizon 不会复现这个收益。可调的杠杆是 milestone 选点质量,而非前瞻长度。
(剂量-反应亦支持:LMWM 成分越重收益越大 —— LaWM +2.0 ns → hintdrop +6.0** → dual2q/机制② +8.5***。)

痛点 2相对空间参照任务(t6):LMWM 用 60% 步数达到同一天花板 中

⚠️ 2026-07-19 二次更正 —— 本节结论已被官方 ckpt 重测推翻并降级

本节原为「相对空间参照任务:LaWM 完全拿不到收益」,依据是我们自训的 armB(LaWM)t6 仅 78.5, 而 LMWM 达 85.0(+8.5, p<0.001)。

痛点 3遮挡任务:LaWM 不只是无收益,而是有害 中

LaWM 差在哪:t9 =「把黄白杯放进微波炉并关门」。目标物进入炉腔后被遮挡, 此时"预测 t+7 帧的观测特征"在数学上是病态的 —— 未来观测由遮挡主导,不可预测。 固定时延无法回避这个窗口,只能强迫模型去拟合噪声。

§2

架构方案

LMWM = CRAVE(零训练 milestone 发现)+ MilestoneGenerator(milestone 条件生成器);作为 LaWAM 世界模型 decoder 的即插即用替身,构成 LMWAM。

LMWM 概览(两阶段)

阶段 1 — CRAVE milestone 发现(零训练):
  冻结 DINOv3-base 编码器 → 提取 episode grid 特征 [N,256,768]
  → 跨 episode gist pooling → 聚类 → 单调 Viterbi readout
  → M 个有序 milestone 质心 + 每帧的 milestone 标签

阶段 2 — Milestone 条件生成器训练:
  (cur_frame_feature, target_code) → MilestoneGenerator(AdaLN) → next_milestone_feature
  损失: L = smooth_L1(gen(g_t, z), g_m+1) + λ_lift·ReLU(cos(gen,g_t) − cos(gen,g_m+1))
  输入对: {(cur_fi, first[milestone+1])} —— 保证跨语义边界

关键属性 —— 编码器无关(CRAVE 只要冻结特征空间有判别力:DINOv3 / SigLIP / VLA 自身编码器)· 自适应时延(milestone 间距由数据决定 2–108 帧,非固定 τ)· 稀疏+高信息(每个训练对跨越至少一个语义阶段边界,下界保证信息密度)。

与 LaWAM 的集成(→ LMWAM)

原 LaWAM:  h_t → [LaWM decoder](h_t, latent_code)   → h_{t+τ}         (τ 固定 0.35/1.2/1.6s)
LMWAM:    h_t → [MilestoneGenerator](h_t, milestone_code) → h_{milestone+1}  (自适应)

最小侵入 swap(高内聚低耦合):① 保留原 LAM 的 extract_vision_features(DINOv3-vitb16,冻结共享);② 只替换 lam.decoder 为 MilestoneGenerator;③ 训练时 h_t1_gt 从 t+τ 改为 milestone+1 帧特征;④ swap_teacher=True:量化器改用 LMWM InverseEnc(保证 code 空间自洽)。

§4

不足与开放问题

以下问题 LMWM 并未解决,部分是两种世界模型共有的。列在此处而非 §1,是因为它们不构成 LMWM 的立论依据。

① 目标歧义下 WM 注入噪声 —— LMWM 同样有

必须诚实的两点

· 没有任何方案的均值差是显著的(全部 |t|≤1.8)——不得声称"WM 伤害 t8"。真实且稳定的效应是方差膨胀:1.9 → 6.4~13.9,方差比 11~53×(F(3,3) 临界 9.3@0.05)。

· LMWM 的 dual2q 方差 11.4,与 LaWM 的 13.9 同一量级 —— 这个问题 LMWM 没有解决。

机理假说(无直接证据):潜在视觉目标无法指明是两个同款壶中的"哪一个",指引因此随 seed 摇摆。 我们尝试用失败模式(超时卡死 vs 早期失败)去检验,没有拿到信号 —— LIBERO 评测不提前终止,100% 的失败都是跑满 550 步,该指标无区分力。 验证需改用轨迹级分析(如目标物切换次数)。

② 聚合收益无法声称

dual2q 94.80±0.71 vs 纯 VLA 94.30±0.54,+0.5pt,小于 SEM。 正贡献(t6 +8.5)被 t8/t9 的负贡献与 8 个饱和任务稀释。 结论:WM 不是普适 VLA 增强,而是场景特异的 —— 只在"目标由相对空间关系定义"时兑现。 LIBERO-10 有 8/10 任务饱和在 93~100,已不足以测量这类机制(外部亦有 LIBERO-Plus 2510.13626 / LIBERO-PRO 2510.03827 指出该基准 "almost saturated")。

③ 机制②(diffusion-t 调度 ms dropout)未证明有效

相对其基座 dual2q:聚合 +0.42(t=0.79, ns),在唯一有效的 t6 上零增量(同为 85.0)。 唯一可见的作用是把 t8 方差从 11.4 压到 8.6 —— 方向上支持"指引门控"的思路,但幅度不足以定论。 门控该怎么做仍是开放问题。

⑤ 世界模型与 VLA 不同特征空间(重做+标定完成 · 倾向"空间重要")

世界模型与 LAM 在冻结 DINOv3 空间,VLA 基座是 Qwen3-VL (取 visual().pooler_output = merger 输出 2048 维 = LLM 实际消费层),二者靠注入 adapter 衔接。 架构定版把「把 LMWM 搬进 VLA 自身编码器空间」列为 P3 终局方向,张力在于 frozen-DINOv3 本就是为跨本体普适性而选。为此做了可证伪的廉价预实验(仅特征提取+分段,无需训练)。
