# Task_A base 数据集是否值得做一次预处理? — 全维度质量扫描

> **结论 (TL;DR)**: **不值得**做去噪 / 平滑 / 再清洗类预处理。base teleop 动作信号在全 19 个日期上均匀干净 (高频噪声 0.01-0.03%、teleport 稀有且已黑名单覆盖、end-snap 几乎不存在)。看似异常的 4-29 gripper 经精确分桶证实是**正常夹布 (tight 3-10mm)**, 非半夹 attractor。真正的杠杆在**数据覆盖 + ckpt 能力**, 不在动作信号清洗。
> **起因**: 真机 EE "大幅来回震荡" 诊断 (见 `../../deployment/inference/realtime_vla/ee_stability_layer1.md` §3.5) 判定为任务运动而非噪声后, 进一步问: 训练数据本身是否含可去除的退化, 值得一次预处理?
> **创建**: 2026-05-28 · **方法可复现**: 见文末数据源 + 指标定义

---

## 1. 方法

- **范围**: `Task_A/base/` 全部 19 个日期 (2026-04-23 ~ 05-28), 每天抽样 15 ep。
- **臂关节** dim [0-5,7-12] (排除 gripper 6/13); **gripper** dim 6(L)/13(R)。30Hz, Nyquist 15Hz。
- **四个"可去除退化"维度**:
  1. **高频噪声预算** = 动作 PSD 中 >5Hz 能量占比 (任务运动 <2Hz; >5Hz = 可被低通安全去除的 jitter)
  2. **病态 teleport** = `|Δq| > 0.5 rad/frame` 的硬跳变 / kFrame (CAN drop / teleport; 阈值校准过 — 0.1rad 会误抓真实快动作)
  3. **end-snap** = 末尾 6 帧某臂关节 >0.3rad 跳变的 ep 数 (record-stop 伪影)
  4. **gripper 分桶 run-length** = mid-range 是短暂过渡还是长时间停驻; 停驻是夹布 (tight) 还是半夹 (loose)

## 3. 逐维度判定

| 维度 | 全集表现 | 判定 |
|---|---|---|
| 高频噪声 (>5Hz) | task 日期 **0.01-0.03%** (仅 5-16 静止 ideal 1.009%, 是不动臂上的微噪声相对占比高, 绝对值极小) | **无可去除 jitter → 低通/平滑处处无用** |
| teleport (>0.5rad) | **0.00-0.30/kf**, 均值 0.12, 极稀有 | 已被 Class C 黑名单 (129 eps, `Task_A/base/analysis/07_classC_blacklist.csv`) 覆盖 |
| end-snap | 抽样全 **0** (全集仅 5 eps, `06_end_snap_trim.csv`) | 极稀有, 已识别 |
| proxOsc° | 3.7-7.1°, 均值 4.7° | 任务节奏一致, 非异常 (见 §3.5 — policy 比人类还小) |

### 3.1 spike 阈值校准 (重要)

0.1rad/frame 阈值会**误抓真实快动作**。4-28 在 `>0.1` 高达 112.9/kf 但 `>0.5` 为 **0** → 那是"简单叠、衣服预铺开"日期的快速 teleop, 不是 CAN drop。真正的病态 teleport 必须用 `>0.5rad` (5-7° 单帧 = 171°/s, 物理上不合理) 来界定, 全集 ≤0.3/kf。

### 3.2 4-29 gripper "异常" — 误报, 实为正常夹布

4-29 的 gmid (3-25mm) = 14% 是全集 2x, 一度怀疑是 §5 半夹 attractor。**精确分桶后否定**:

| date·dim | full<3 | **tight 3-10** | **loose 10-25** | mid 25-50 | open>50 |
|---|---:|---:|---:|---:|---:|
| 4-29 R | 55.0 | **12.8** | **0.9** | 12.3 | 19.0 |
| 4-29 L | 44.8 | **9.6** | **2.5** | 16.6 | 26.5 |
| 5-27 R (clean) | 68.2 | 1.1 | 0.8 | 10.0 | 19.8 |
| 5-18 R | 63.0 | 0.6 | 0.6 | 8.0 | 27.8 |

- 4-29 那 14% 几乎全在 **tight 3-10mm (夹紧布)**, loose 10-25mm **只 0.9%**。
- 逐 ep: 高-mid 的 ep 是长 hold (6-16s) 停在 **4-8mm, std 1-2.8mm** = 实打实夹着布在叠 (布厚几 mm)。
- 与 §5 的 **25-30mm 松悬半夹 attractor** 是不同 regime。4-29 反而是 firm-grasp 的**好示范** (5-27/5-18 的 tight 仅 0.6-1.1%, 更多直接闭到 0)。
- **误报根因**: 之前用 3-25mm 一个大桶, 把"夹紧布(3-10)"和"松半夹(10-25)"混了。

## 4. 已有的预处理成果 (无需重做)

`Task_A/base/analysis/` 下已有上一轮针对性清理产物:
- `07_classC_blacklist.csv` — 129 eps (|Δ|>0.5rad mid-spike, CAN drop)
- `06_end_snap_trim.csv` — 5 eps 末尾归零
- 5-18~5-22 R-grip 传感器零点漂移 — 已非线性 rescale 修复 (Curve B, 见 `00_action_only_finetune_history` 相关)

唯一**值得核实**的点: 训练 config 是否真的**排除了** Class C 黑名单那 129 ep (这是已有预处理可能没生效的环节)。

> **2026-06-03 构建管线更新 — norm_stats 自动化**: 数据集构建脚本(`train_scripts/kai/data/build_no_release.py`)现**在构建完成后自动从刚建好的数据集(re)算 norm_stats**(新 helper `norm_stats_from_dataset.py`,与 `kai0/scripts/compute_norm_states_fast.py` 数值字节级一致、但 config-free)。意义:① 不再需要手动跑 `compute_norm_stats`(消除"忘算"这一头号提交坑,见 `../../deployment/training_ops/submission/training_pitfalls_common.md §1`);② 杜绝误用**源数据集**的旧 norm_stats(裁剪/合并后分布已变)。约束:仅适用 kai pi0/pi05(`--action-dim` 默认 32);EE6D/xvla 集靠 loss-scale 不归一、不适用;构建须用 kai0 venv(openpi 依赖)。`--no-norm-stats` 可关。

## 5. 净结论

- ❌ 去噪 / 平滑 / 再清洗类预处理 **不值得**: 数据已干净, 且会误伤真实快动作 (04-28) + firm grasp (4-29) + 任务节奏 — 这些都是**信号不是噪声**。
- ✅ 真正杠杆: **数据覆盖** (成功抓取 / init-pose 覆盖) + **ckpt 能力**, 属数据采集 / 训练侧, 不在动作信号清洗。
- 关联部署侧诊断: `../../deployment/inference/realtime_vla/ee_stability_layer1.md` §3.4 (RTC×EMA 频域) + §3.5 (震荡=任务运动)。
